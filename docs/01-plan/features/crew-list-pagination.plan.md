## 1. Overview

### 1.1 Purpose

`CrewList.tsx`에서 렌더링하는 크루 목록을 페이지당 10개로 제한하고, 하단에 페이지 번호 버튼을 표시하여 사이드바 스크롤 문제를 해결한다.

### 1.2 Background

랭킹 섹션(TOP 5) 추가 후 사이드바 콘텐츠가 더 길어졌다. 크루가 10개 이상일 경우 사이드바 전체 스크롤이 발생하여 지도와 목록을 동시에 활용하기 어렵다. 백엔드 변경 없이 클라이언트 슬라이싱으로 해결한다.

### 1.3 Related Documents

- 크루 목록 UI: `frontend/src/components/CrewList.tsx`
- 상위 상태 관리: `frontend/src/App.tsx`

## 2. Scope

### 2.1 In Scope

- `CrewList.tsx` 내부에 `currentPage` 상태 추가
- 필터링된 `crews` 배열을 페이지 기준으로 슬라이싱 (`slice`)
- 페이지 번호 버튼 UI (❮ 이전 / 1 2 3 ... / 다음 ❯)
- 필터(난이도·요일) 또는 검색어 변경 시 `currentPage → 1` 자동 리셋
- 결과 수 표시를 전체 개수 기준으로 유지 (`crews.length`개 크루)

### 2.2 Out of Scope

- 서버 사이드 페이지네이션 (백엔드 API 변경 없음)
- 무한 스크롤
- 페이지당 표시 개수 사용자 설정
- URL에 페이지 번호 반영

## 5. Technical Considerations

### 5.1 페이지네이션 로직

```
PAGE_SIZE = 10
totalPages = Math.ceil(crews.length / PAGE_SIZE)
pagedCrews = crews.slice((currentPage - 1) * PAGE_SIZE, currentPage * PAGE_SIZE)
```

### 5.2 상태 관리 위치

`currentPage` 상태를 `CrewList.tsx` 내부에서 관리한다 (`useState`).
필터·검색은 props(`filterLevel`, `filterDay`, `searchQuery`)로 전달되므로
`useEffect`로 이 값 변화를 감지하여 `setCurrentPage(1)` 호출.

### 5.3 페이지 번호 버튼 표시 범위

페이지가 많을 때 전부 표시하면 UI가 넘침. 현재 페이지 기준 ±2 윈도우로 표시:
- 총 5개 미만이면 전부 표시
- 총 5개 이상이면 현재 페이지 중심으로 최대 5개 표시

## 6. Risks

| 위험 | 영향 | 대응 |
|------|------|------|
| 선택된 크루가 다른 페이지에 있을 때 | 하이라이트 표시는 되지만 보이지 않음 | 현재 선택 크루가 속한 페이지로 자동 이동 고려 (Should) |
| 크루 수가 10개 이하일 때 | 페이지네이션 UI 불필요하게 표시 | FR-08: 1페이지만 있을 때 숨김 처리 |

## 7. Success Criteria

- [ ] 크루 목록이 10개씩 분할되어 표시됨

- [ ] 첫/마지막 페이지에서 이전/다음 버튼 비활성화
- [ ] 필터·검색 변경 시 1페이지로 자동 리셋
- [ ] 총 1페이지일 때 페이지네이션 UI 미표시
- [ ] 결과 수는 전체 기준 표시 유지
